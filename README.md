# Steel Plate Fault Database  
데이터베이스 과제 (Database Assignment)

---

## 1. 프로젝트 개요

본 프로젝트는 데이터베이스(Database)를 웹 애플리케이션(Web Application)에 연동하여 사용하는 방법을 학습하기 위한 과제이다.

산업 공정 중 수집된 철판(Steel Plate) 검사 데이터를 기반으로, 결함 유형(Fault)과 수치형 검사 결과를 데이터베이스에 저장하고, 사용자가 다양한 조건으로 데이터를 조회할 수 있도록 구현하였다.

---

## 2. 프로젝트 목적

이 프로젝트의 주요 목적은 다음과 같다.

- 관계형 데이터베이스 구조 설계 및 정규화
- 여러 테이블 간 관계(FOREIGN KEY)를 활용한 데이터 조회
- SQL JOIN 및 WHERE 조건을 활용한 데이터 필터링
- 사용자 입력에 따른 동적 SQL 조건 구성
- 데이터베이스 기반 웹 애플리케이션 구현 경험

---

## 3. 데이터 개요

### 데이터 출처
- UCI Machine Learning Repository의 Steel Plate Fault 데이터셋을 기반으로 한다.
- 원본 데이터를 정리하고 구조를 재설계하여 데이터베이스에 저장한다.

### 데이터 특징
- 각 철판은 하나의 결함 유형(Fault)을 가진다.
- 각 철판에 대해 여러 수치형 검사 값이 존재한다.
- 결함 정보와 수치 정보는 서로 다른 테이블로 분리하여 저장한다.

---

## 4. 데이터베이스 구조

### 4.1 faults 테이블

결함 정보 테이블은 철판에서 발생할 수 있는 결함 유형을 관리하기 위한 기준 테이블이다.

각 결함은 고유한 식별 코드를 통해 관리되며,
결함의 명칭과 해당 결함에 대한 설명 정보를 포함한다.
이 테이블은 여러 철판에서 동일한 결함 정보를 참조할 수 있도록 설계되었다.

---

### 4.2 plates_new 테이블

철판 정보 테이블은 개별 철판을 식별하기 위한 기본적인 정보를 저장하는 테이블이다.

각 철판은 고유한 식별자를 가지며,
해당 철판에 발생한 결함 유형을 결함 정보 테이블의 식별 코드를 통해 참조한다.

---

### 4.3 plate_measurements 테이블

철판의 수치형 검사 데이터를 저장하는 테이블이다.

철판 검사 수치 테이블은 철판의 물리적 검사 결과를 저장하기 위한 테이블이다.

두께, 폭, 표면적과 같은 수치형 데이터가 이 테이블에 저장되며,
해당 데이터는 범위 기반 필터링과 조건 검색에 활용된다.

---

### 4.4 정규화

각 테이블은 하나의 개념만을 표현하도록 설계되었으며,
비주요 속성은 기본키에만 함수적으로 종속되도록 구성하였다.

이와 같은 설계를 통해 데이터 중복 및 갱신 이상을 방지할 수 있으며,
본 데이터베이스 구조는 **BCNF (Boyce–Codd Normal Form)**를 만족한다.

---

## 5. 구현 기능

---

### 5.1 철판 목록 조회 (/plates)

- 모든 철판 데이터를 페이지 단위로 조회할 수 있다.
- 결함 유형(Fault)을 기준으로 철판 목록을 필터링할 수 있다.
- 여러 테이블을 JOIN하여 필요한 정보를 함께 조회한다.

---

### 5.2 범위 기반 필터링 (/plates/filter)

철판의 수치형 검사 데이터를 기준으로 철판 목록을 추가로 필터링할 수 있다.

#### 지원하는 필터 조건
- Thickness (두께)
- Width (폭)
- Surface Area (표면적)

각 값에 대해 최소값과 최대값을 설정할 수 있으며,
입력된 조건만 SQL 질의에 반영되어 페이지 상단에 적용한 조건이 표시되도록 구현하였다.

---

### 5.4 철판 상세 정보 조회 (/plates/<plate_id>)

- 특정 철판 하나에 대한 상세 정보를 조회할 수 있다.
- 결함 정보와 수치형 검사 데이터를 함께 확인할 수 있다.
- 테이블 간 관계를 실제 조회 결과를 통해 확인할 수 있다.
